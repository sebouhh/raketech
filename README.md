# RakeTech

A modular monolith built with TypeScript, Next.js 15, tRPC, Drizzle ORM, and deployed on Vercel.

See [ADR-001](/RAK/issues/RAK-2#document-plan) for the full architecture decision record.

---

## Prerequisites

- **Node.js** 22 LTS ([nvm](https://github.com/nvm-sh/nvm) recommended: `nvm use`)
- **pnpm** 9+ (`npm install -g pnpm`)
- **Git**

---

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/sebouhh/raketech.git raketech
cd raketech

# 2. First-time setup (installs deps + creates .env.local)
make setup

# 3. Fill in real values in apps/web/.env.local
#    (see .env.example for all required variables)
cp .env.example apps/web/.env.local
# Edit apps/web/.env.local with real credentials

# 4. Start the development server
make dev
```

Open [http://localhost:3000](http://localhost:3000).

## Repository Structure

```
raketech/
├── apps/
│   └── web/               # Next.js 15 app (UI + tRPC API)
├── packages/
│   ├── db/                # Drizzle ORM schema, migrations, client
│   ├── ui/                # Shared React component library
│   └── config/            # Shared TypeScript configs
├── .github/
│   └── workflows/
│       ├── ci.yml         # Lint, typecheck, test on every PR
│       ├── deploy.yml     # Deploy to Vercel on merge to main
│       └── e2e.yml        # Playwright E2E on every PR
├── .devcontainer/         # VS Code Dev Container config
├── Makefile               # Common dev commands
├── biome.json             # Linter + formatter config
├── turbo.json             # Turborepo task pipeline
└── pnpm-workspace.yaml
```

## Common Commands

| Command | Description |
|---|---|
| `make dev` | Start all dev servers |
| `make build` | Build all packages and apps |
| `make lint` | Run Biome linter |
| `make lint-fix` | Fix lint issues automatically |
| `make typecheck` | TypeScript type check |
| `make test` | Run Vitest unit/integration tests |
| `make test-e2e` | Run Playwright E2E tests |
| `make db-generate` | Generate new DB migration from schema changes |
| `make db-migrate` | Apply pending migrations to the database |
| `make db-push` | Push schema directly (dev only, no migration file) |
| `make db-studio` | Open Drizzle Studio (visual DB browser) |
| `make clean` | Remove all build artifacts |

## Environment Variables

Copy `.env.example` to `apps/web/.env.local` and fill in the values.

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Neon PostgreSQL connection string |
| `UPSTASH_REDIS_REST_URL` | Yes | Upstash Redis REST URL |
| `UPSTASH_REDIS_REST_TOKEN` | Yes | Upstash Redis REST token |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Yes | Clerk publishable key |
| `CLERK_SECRET_KEY` | Yes | Clerk secret key |
| `R2_ACCOUNT_ID` | No (file uploads) | Cloudflare R2 account ID |
| `RESEND_API_KEY` | No (email) | Resend API key |
| `NEXT_PUBLIC_POSTHOG_KEY` | No (analytics) | PostHog project key |
| `NEXT_PUBLIC_SENTRY_DSN` | No (errors) | Sentry DSN |

## External Services Setup

Before running locally you need accounts on:

1. **[Neon](https://neon.tech)** — create a project, copy the connection string
2. **[Clerk](https://clerk.com)** — create an app, copy publishable + secret keys
3. **[Upstash](https://upstash.com)** — create a Redis database, copy REST URL + token

All other services (Sentry, PostHog, Resend, R2) are optional for local dev.

## Database Workflow

```bash
# After changing packages/db/src/schema/*.ts:
make db-generate   # Creates a migration file in packages/db/migrations/

# Apply migrations to your local / staging DB:
make db-migrate

# Browse data visually:
make db-studio
```

## CI/CD

- **Every PR**: lint + typecheck + unit tests + build check + E2E tests
- **Merge to main**: auto-deploys to Vercel production, then runs DB migrations

Required GitHub repository secrets:

```
VERCEL_TOKEN
DATABASE_URL
DATABASE_URL_TEST
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY_TEST
CLERK_SECRET_KEY_TEST
```
